# LocJSON

API-сервис для управления проектами локализации контента: загрузка документов с ключами для перевода, работа с несколькими языками (source/target), назначение переводчиков (фрилансеров) на проекты и импорт готовых переводов.

## Стек

- PHP 8.4, Laravel 13
- MySQL 8.4
- Redis
- Laravel Sanctum (авторизация по токену)
- Docker / Docker Compose (nginx + php-fpm)

## Запуск проекта

1. Склонировать репозиторий и перейти в папку `src`, скопировать `.env.example` в `.env` (если ещё не создан).

2. Поднять контейнеры:

```bash
docker-compose up -d --build
```

3. Зайти в контейнер приложения и установить зависимости:

```bash
docker-compose exec app sh
composer install
php artisan key:generate
php artisan migrate --seed
```

4. Проект будет доступен на `http://localhost:8000`.

## Аутентификация

Регистрация и вход выдают Bearer-токен (Sanctum), который передаётся в заголовке:

```
Authorization: Bearer {token}
```

## Основные сущности

### Account
Регистрация, вход и получение данных текущего аккаунта. Тип аккаунта: `individual`, `freelancer` или `ltd` (для юрлиц указывается `companyName`).

### Users
Поиск пользователей, в частности фрилансеров-переводчиков (`/v1/users/freelancers`) — с фильтром по имени и пагинацией.

### Languages
Справочник доступных языков (`id`, `name`, `locale`).

### Projects
CRUD для проектов локализации. У каждого проекта есть исходный язык (`source`) и один или несколько целевых (`target`), настройки (например, `useMachineTranslate`), список документов и назначенных переводчиков, а также общий прогресс перевода.

### Documents
Документы — это наборы пар «ключ / значение» (обычно из JSON-файлов локализации), привязанные к проекту. Поддерживается:
- добавление одного или нескольких документов сразу,
- получение списка документов проекта и содержимого конкретного документа на нужном языке (`?lang=`),
- импорт переводов по ключам,
- удаление документа.

### Team
Назначение переводчика (`performerId`) на проект и получение списка проектов, назначенных текущему пользователю.

## API-коллекция

В репозитории есть Postman-коллекция (`LocJSON_postman_collection.json`) со всеми эндпоинтами, примерами тела запроса и ответов — самый быстрый способ разобраться в API.
